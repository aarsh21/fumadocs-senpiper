# Part 1: Universal Properties

These properties apply to **all field types**.

## Identity & Display

### title
**Type:** `string`  
**Required:** Yes  
**Category:** Identity

The human-readable label displayed to users.

```json
{ "title": "Customer Name" }
```

---

### type
**Type:** `string`  
**Required:** Yes  
**Category:** Identity  
**Allowed Values:** `"string"`, `"number"`, `"boolean"`, `"object"`, `"array"`

The data type determining how the value is stored. See [Field Types](concepts/field-types.md).

| Value | Storage | Example Answer |
|-------|---------|----------------|
| `string` | Text | `"John Doe"` |
| `number` | Numeric | `42` or `3.14` |
| `boolean` | True/False | `true` |
| `object` | Nested structure | `{ "city": "Mumbai" }` |
| `array` | List | `[{ ... }, { ... }]` |

```json
{ "type": "string" }
```

---

### description
**Type:** `string`  
**Required:** Yes  
**Category:** Identity  
**Allowed Values:** `textfield`, `email`, `qrcode`, `barcode`, `number`, `phone`, `label`, `location`, `progress`, `rating`, `network`, `group_member`, `string_list`, `poll`, `debit`, `multimedia`, `section`, `form`, `richtext`, `timestamp`, `time`, `date`, `duration`, `business_card`, `report`, `checklist`, `geofence`, `separated_input`, `object_detection`, `object_dimension`, `text_extraction`

The UI component type to render. See [Field Types Reference](field-types/README.md).

**Common categories:**
- **Text:** `textfield`, `email`, `phone`, `richtext`
- **Selection:** `string_list`
- **Number:** `number`, `rating`, `progress`
- **Date/Time:** `timestamp`, `date`, `time`, `duration`
- **Media:** `multimedia`
- **Location:** `location`, `geofence`
- **Scanner:** `qrcode`, `barcode`
- **Structure:** `section`, `form`
- **Display:** `label`

```json
{ "description": "textfield" }
```

---

### hint
**Type:** `string`  
**Required:** No  
**Category:** Appearance  
**Related:** [Field Appearance](concepts/field-appearance.md)

Help text displayed near the field to guide users. Shown as tooltip or inline based on `inlineHint`.

```json
{ "hint": "Enter your full name as it appears on your ID" }
```

---

### inlineHint
**Type:** `boolean`  
**Required:** No  
**Default:** `false`  
**Category:** Appearance

When `true`, displays hint text inline below the field instead of as a tooltip icon.

```json
{
  "hint": "We'll send a verification email to this address",
  "inlineHint": true
}
```

---

### placeholder
**Type:** `string`  
**Required:** No  
**Category:** Appearance

Placeholder text shown inside empty input fields.

```json
{ "placeholder": "e.g., john.doe@example.com" }
```

---

### layout
**Type:** `string`  
**Required:** No  
**Category:** Appearance  
**Related:** [Field Appearance](concepts/field-appearance.md)

Visual variant of the field component. Available values depend on field type:

**For `string_list` (selection fields):**
| Value | Renders As |
|-------|------------|
| *(default)* | Dropdown picker |
| `radio` | Radio buttons |
| `checkbox` | Checkboxes (multi-select) |
| `small_pill` / `smallPill` | Small pill/chip buttons |
| `large_pill` / `largePill` | Large pill/chip buttons |

**For `section` (containers):**
| Value | Renders As |
|-------|------------|
| *(default)* | Grouped fields |
| `newPage` | Renders on separate screen |
| `accordian` | Collapsible accordion |
| `expandable` / `expanded` | Initially expanded |
| `collapsible` / `collapsed` | Initially collapsed |
| `inline` | Inline with parent |
| `book` | Book-style pagination |
| `tab` | Tab navigation |
| `table` | Table layout (for arrays) |
| `card` | Card container |

**For `label` (display-only):**
| Value | Renders As |
|-------|------------|
| `h1`, `h2`, `h3` | Heading styles |
| `info` | Info callout (blue) |
| `warn` | Warning callout (yellow) |
| `error` | Error callout (red) |
| `mediaOnly` | Only show media, no text |

**For `number`:**
| Value | Renders As |
|-------|------------|
| `counter` | Stepper with +/- buttons |

```json
{ "layout": "radio" }
```

---

### webLayout
**Type:** `string`  
**Required:** No  
**Category:** Appearance  
**Allowed Values:** `small_pill`, `large_pill`, `accordian`, `radio`, `checkbox`, `table`

Layout variant specifically for web rendering. Takes precedence over `layout` on web platform.

```json
{
  "layout": "newPage",
  "webLayout": "table"
}
```

---

### wrapStyle
**Type:** `string`  
**Required:** No  
**Category:** Appearance  
**Allowed Values:** `"trim"`, `"wrap"`

How to handle long text in title and content.

| Value | Behavior |
|-------|----------|
| `trim` | Truncate with ellipsis (...) |
| `wrap` | Wrap to multiple lines |

```json
{ "wrapStyle": "wrap" }
```

**Required:** No

```json
  "description": "number"
```


---

### renderAsMd
**Type:** `boolean`  
**Required:** No  
**Default:** `false`  
**Category:** Appearance

When `true`, renders field content as Markdown.


## Structure (Objects & Arrays)

### properties
**Type:** `Map<string, SchemaProperty>`  
**Required:** For `type: "object"`  
**Category:** Structure

Child field definitions for object types. Keys become field identifiers in the answer.

```json
{
  "type": "object",
  "description": "section",
  "properties": {
    "firstName": { "title": "First Name", "type": "string", "description": "textfield" },
    "lastName": { "title": "Last Name", "type": "string", "description": "textfield" }
  }
}
```

---

### items
**Type:** `object` (contains `properties`, `order`, `required`)  
**Required:** For `type: "array"` sections  
**Category:** Structure

Schema definition for each element in an array (repeating section).

> ⚠️ **Important:** For array sections (`type: "array"`), child fields are defined inside `items.properties`, NOT in `properties` directly. This follows JSON Schema array convention.

**Structure:**
```json
{
  "type": "array",
  "description": "section",
  "items": {
    "type": "object",
    "properties": {
      "fieldKey": { /* SchemaProperty */ }
    },
    "order": ["fieldKey1", "fieldKey2"],
    "required": ["fieldKey1"]
  }
}
```

**Complete Example:**
```json
{
  "title": "Family Members",
  "type": "array",
  "description": "section",
  "minRows": 1,
  "maxRows": 10,
  "newRowLabel": "Add Family Member",
  "items": {
    "type": "object",
    "properties": {
      "name": { 
        "title": "Name", 
        "type": "string", 
        "description": "textfield" 
      },
      "relation": {
        "title": "Relation",
        "type": "string",
        "description": "string_list",
        "enum": ["Spouse", "Child", "Parent", "Sibling"]
      },
      "age": {
        "title": "Age",
        "type": "number",
        "description": "number"
      }
    },
    "order": ["name", "relation", "age"],
    "required": ["name", "relation"]
  }
}
```

**Comparison: Object Section vs Array Section**

| Aspect | Object Section (`type: "object"`) | Array Section (`type: "array"`) |
|--------|-----------------------------------|--------------------------------|
| Child properties location | `properties` | `items.properties` |
| Child order location | `order` | `items.order` |
| Child required location | `required` | `items.required` |
| UI Rendering | Single group of fields | Repeatable rows |

---

### order
**Type:** `string[]`  
**Required:** No (but recommended)  
**Category:** Structure

Display order of child fields. Fields not in order may be hidden or appear last.

```json
{ "order": ["firstName", "lastName", "email", "phone"] }
```

---

### required
**Type:** `string[]`  
**Required:** No  
**Category:** Structure

List of required child field keys (JSON Schema compatibility). Prefer using `accessMatrix.mandatory` for fine-grained control.

```json
{ "required": ["firstName", "email"] }
```

Field-level access control for visibility, editability, and requirements.

```json
{
  "accessMatrix": {

  }
```

## Conditional Logic

### predicates
**Type:** `Predicate[]`  
**Required:** No  
**Category:** Conditional Logic  
**Related:** [Conditional Logic](concepts/conditional-logic.md)

Conditional rules that control field behavior dynamically.

```json
{
  "predicates": [{
    "condition": "this.status == 'Active'",
    "action": "APPLY_ACCESS_MATRIX",
    "actionConfig": {
      "accessMatrix": {
        "visibility": "VISIBLE",
        "mandatory": true
      }
    }
  }]
}
```

See [Predicate Object](#predicate-object) for all properties.

---

### dependentKeys
**Type:** `string[]`  
**Required:** No  
**Category:** Conditional Logic  
**Related:** [Conditional Logic](concepts/conditional-logic.md)

Fields that should re-evaluate their predicates when this field changes. Essential for cascade behavior.

```json
{
  "title": "State",
  "dependentKeys": ["district", "city", "fullAddress"]
}
```

When State changes → predicates on District, City, and FullAddress are re-evaluated.

---

### formulaKeys
**Type:** `string[]`  
**Required:** No  
**Category:** Conditional Logic  
**Related:** [Calculated Fields](recipes/calculated-field.md)

Fields used in this field's CALC formula. Declares dependencies for calculation triggering.

```json
{
  "title": "Total Amount",
  "formulaKeys": ["price", "quantity", "discount"],
  "predicates": [{
    "action": "CALC",
    "actionConfig": { 
      "formula": "(this.price * this.quantity) - this.discount" 
    }
  }]
}
```

---

### dependentResponseConfigurations
**Type:** `Map<string, List<Integer>>`  
**Required:** No  
**Category:** Conditional Logic

Advanced configuration for dependent response behavior.

## Validation

### pattern
**Type:** `string` (regex)  
**Required:** No  
**Category:** Validation

Regular expression for validating string input.

```json
{
  "pattern": "^[A-Z]{5}[0-9]{4}[A-Z]{1}$",
  "validationMessage": "Enter valid PAN (e.g., ABCDE1234F)"
}
```

---

### validationMessage
**Type:** `string`  
**Required:** No  
**Category:** Validation

Custom error message shown when validation fails.

```json
{
  "pattern": "^[a-zA-Z ]+$",
  "validationMessage": "Name can only contain letters and spaces"
}
```

---

### minimum / maximum
**Type:** `number`  
**Required:** No  
**Category:** Validation

**For numbers:** Min/max allowed value.

```json
{
  "type": "number",
  "minimum": 0,
  "maximum": 100
}
```

**For dates:** Offset from current date (combined with `timeUnit`).

```json
{
  "description": "timestamp",
  "minimum": -7,
  "maximum": 30,
  "timeUnit": "DAY"
}
```
↑ Allows dates from 7 days ago to 30 days from now.

---

### minSize / maxSize
**Type:** `integer`  
**Required:** No  
**Category:** Validation  
**Constraints:** 1 to MAX_MEDIA_UPLOAD_SIZE

**For strings:** Min/max character count.
**For multimedia:** Min/max file size in bytes.

```json
{
  "description": "multimedia",
  "maxSize": 5242880
}
```
↑ 5MB limit

---

### minRows / maxRows
**Type:** `integer`  
**Required:** No  
**Default:** `minRows: 1`, `maxRows: unlimited`  
**Category:** Section/Array  
**Constraints:** 0 to 999

For arrays: Min/max number of rows allowed.

- `minRows`: Number of placeholder items shown by default (default: 1)
- `maxRows`: Maximum items allowed (default: unlimited)

```json
{
  "type": "array",
  "description": "section",
  "minRows": 1,
  "maxRows": 10
}
```

> **Note:** By default, repeating sections show 1 placeholder item so users have a form to fill immediately.

# Part 4: Nested Objects

Detailed reference for complex nested objects.

## AccessMatrix Object

Controls field-level access and visibility. See [Visibility & Access](concepts/visibility-access.md).

| Property | Type | Values | Purpose |
|----------|------|--------|---------|
| `visibility` | string | `VISIBLE`, `INVISIBLE`, `GONE` | Fill mode visibility |
| `viewModeVisibility` | string | `VISIBLE`, `INVISIBLE`, `GONE` | View mode visibility |
| `mandatory` | boolean | true/false | Required for submission |
| `readOnly` | boolean | true/false | Prevent editing |
| `roles` | string[] | `ADMIN`, `SUPERADMIN`, `USER`, `INITIATOR`, `ADHOC`, `CDR` | Role-based access |
| `cdr` | string[] | Custom role names | Company-defined roles |
| `userIds` | UUID[] | User IDs | User-specific access |
| `answer` | any | Default value | Pre-populated value |
| `defaultMediaName` | string | Filename | Default media filename |
| `userProfileDriven` | boolean | true/false | Auto-fill from user profile |

**Example:**
```json
{
  "accessMatrix": {
    "visibility": "VISIBLE",
    "mandatory": true,
    "readOnly": false,
    "roles": ["USER", "ADMIN"],
    "answer": "Default Value"
  }
}
```

## Predicate Object

Conditional rules for dynamic behavior. See [Conditional Logic](concepts/conditional-logic.md).

| Property | Type | Purpose |
|----------|------|---------|
| `condition` | string | JavaScript expression (returns boolean) |
| `action` | string | Action type |
| `actionConfig` | object | Action-specific configuration |
| `skipPredicateExecutionOnClient` | boolean | Only run on server |
| `skipPredicateExecutionOnServer` | boolean | Only run on client |

**Action Types:**
- `CALC` - Calculate field value
- `OPTION_FILTER` - Filter selection options
- `APPLY_ACCESS_MATRIX` - Change visibility/mandatory/readOnly
- `VALIDATE` - Custom validation
- `COPY` - Copy value from another field
- `CONDITIONAL_FORMAT` - Apply styling
- `ASYNC_CALC` - Server-side calculation
- `APPEND` - Append to value
- `GEO_FENCE` - Geo-fencing validation
- `FACE_MATCH` - Face matching
- `CUSTOM_FUNCTION` - Custom function call
- `JSON_CALC` - JSON calculation

**Example:**
```json
{
  "predicates": [{
    "condition": "this.country == 'India'",
    "action": "APPLY_ACCESS_MATRIX",
    "actionConfig": {
      "accessMatrix": {
        "visibility": "VISIBLE",
        "mandatory": true
      }
    }
  }]
}
```
