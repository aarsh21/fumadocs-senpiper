## Layout Properties

### `layout` üñ•Ô∏è

Changes the visual representation of a field without changing its data type. Think of it as "skins" for fields.

```json
{
  "fieldKey": {
    "description": "string_list",
    "layout": "radio"
  }
}
```

#### Available Values by Field Type

| Field Type | Layout Options | Description |
|------------|---------------|-------------|
| **string_list** | `radio` | Radio buttons (single select) |
| | `checkbox` | Checkboxes (multi-select) |
| | `small_pill` / `smallPill` | Small pill/chip buttons |
| | `large_pill` / `largePill` | Large pill/chip buttons |
| | *(default)* | Dropdown picker |
| **section** | `newPage` | Renders section on a new screen |
| | `accordian` | Collapsible accordion |
| | `expandable` / `expanded` | Initially expanded |
| | `collapsible` / `collapsed` | Initially collapsed |
| | `inline` | Inline with parent |
| | `book` | Book-style pagination |
| | `tab` | Tab navigation |
| | `table` | Table layout |
| | `card` | Card style |
| **label** | `h1` | Large heading |
| | `h2` | Medium heading |
| | `h3` | Small heading |
| | `info` | Info style (blue) |
| | `warn` | Warning style (yellow) |
| | `error` | Error style (red) |
| | `mediaOnly` | Display only media, no text |
| **number** | `counter` | Stepper/counter UI with +/- buttons |

**Example: Radio Layout**
```json
{
  "gender": {
    "title": "Gender",
    "type": "string",
    "description": "string_list",
    "layout": "radio",
    "enum": ["Male", "Female", "Other"]
  }
}
```

**Example: Accordion Section**
```json
{
  "addressSection": {
    "title": "Address Details",
    "type": "object",
    "description": "section",
    "layout": "accordian",
    "properties": { ... }
  }
}
```

### `webLayout` üñ•Ô∏è

Web-specific layout override. Takes precedence over `layout` when rendering on web.

```json
{
  "items": {
    "description": "section",
    "type": "array",
    "layout": "newPage",
    "webLayout": "table"
  }
}
```

| Value | Description |
|-------|-------------|
| `table` | Render array section as table |
| `small_pill` | Pill selection on web |
| `large_pill` | Large pill selection |
| `accordian` | Accordion on web |
| `radio` | Radio buttons |
| `checkbox` | Checkboxes |

## Text & Label Properties

### `title` üñ•Ô∏è

The field's display label shown to users.

```json
{
  "firstName": {
    "title": "First Name",
    "type": "string"
  }
}
```

### `hint` üñ•Ô∏è

Tooltip or helper text shown near the field. Provides additional context.

```json
{
  "panNumber": {
    "title": "PAN Number",
    "hint": "Enter your 10-character PAN (e.g., ABCDE1234F)",
    "type": "string"
  }
}
```

### `inlineHint` üñ•Ô∏è

When `true`, displays the hint inline with the field instead of as a tooltip.

```json
{
  "email": {
    "title": "Email",
    "hint": "We'll send verification to this email",
    "inlineHint": true
  }
}
```

### `placeholder` üñ•Ô∏è

Placeholder text shown inside empty input fields.

```json
{
  "searchQuery": {
    "title": "Search",
    "placeholder": "Type to search...",
    "type": "string"
  }
}
```

### `prefixText` üñ•Ô∏è

Text displayed before the input (inside the field container).

```json
{
  "price": {
    "title": "Price",
    "prefixText": "‚Çπ",
    "type": "number"
  }
}
```

### `suffixText` üñ•Ô∏è

Text displayed after the input (inside the field container).

```json
{
  "weight": {
    "title": "Weight",
    "suffixText": "kg",
    "type": "number"
  }
}
```

### `wrapStyle` üñ•Ô∏è

Controls how text wraps within the field.

| Value | Description |
|-------|-------------|
| `trim` | Truncate long text with ellipsis |
| `wrap` | Wrap text to multiple lines |

```json
{
  "description": {
    "title": "Description",
    "wrapStyle": "wrap",
    "type": "string"
  }
}
```

### `renderAsMd` üñ•Ô∏è

When `true`, renders label text as Markdown.

```json
{
  "instructions": {
    "title": "**Important:** Please read carefully",
    "description": "label",
    "renderAsMd": true
  }
}
```

### `cameraHint` üñ•Ô∏è

Text displayed when camera is opened for multimedia fields.

```json
{
  "idCard": {
    "title": "ID Card Photo",
    "description": "multimedia",
    "cameraHint": "Position ID card within the frame"
  }
}
```

## Display Configuration Objects

These objects provide fine-grained control over field appearance in different modes.

### `titleDisplayConfiguration` üñ•Ô∏è

Controls how the field title/label is displayed.

```json
{
  "fieldKey": {
    "titleDisplayConfiguration": {
      "buttonConfiguration": {
        "localisationMap": {}
      },
      "backgroundColor": "#f5f5f5",
      "fontColor": "#333333",
      "fontSize": 14,
      "fontWeight": "bold"
    }
  }
}
```

### `viewModeConfiguration` üñ•Ô∏è

Controls appearance when form is in **view/read-only mode**.

```json
{
  "amount": {
    "viewModeConfiguration": {
      "backgroundColor": "#e8f5e9",
      "fontColor": "#2e7d32",
      "fontSize": 16,
      "fontWeight": "bold"
    }
  }
}
```

### `fillModeConfiguration` üñ•Ô∏è

Controls appearance when form is in **edit/fill mode**.

```json
{
  "criticalField": {
    "fillModeConfiguration": {
      "backgroundColor": "#fff3e0",
      "borderColor": "#ff9800",
      "borderWidth": 2
    }
  }
}
```

### `formDisplayConfiguration` üñ•Ô∏è

General display configuration for the field.

```json
{
  "highlighted": {
    "formDisplayConfiguration": {
      "backgroundColor": "#fffde7",
      "borderColor": "#fbc02d"
    }
  }
}
```

### `rowHeaderDisplayConfiguration` üñ•Ô∏è

For array sections, controls how row headers appear.

```json
{
  "items": {
    "type": "array",
    "rowHeaderDisplayConfiguration": {
      "backgroundColor": "#e3f2fd",
      "fontWeight": "bold"
    }
  }
}
```

### FieldDisplayConfiguration Properties

All display configuration objects support these properties:

| Property | Type | Description |
|----------|------|-------------|
| `backgroundColor` | `string` | Background color (hex: `#ffffff`) |
| `fontColor` / `textColor` | `string` | Text color (hex) |
| `borderColor` | `string` | Border color (hex) |
| `borderWidth` | `number` | Border width in pixels |
| `borderRadius` | `number` | Corner radius in pixels |
| `fontSize` | `number` | Font size in pixels |
| `fontWeight` | `string` | `normal`, `bold`, `100`-`900` |
| `fontStyle` | `string` | `normal`, `italic` |
| `padding` | `number` | Internal padding |
| `margin` | `number` | External margin |

## Visibility & Access

### `accessMatrix` ‚öôÔ∏è

Controls field visibility, editability, and requirements.

```json
{
  "sensitiveField": {
    "accessMatrix": {
      "visibility": "VISIBLE",
      "mandatory": true,
      "readOnly": false
    }
  }
}
```

#### AccessMatrix Properties

| Property | Type | Values | Description |
|----------|------|--------|-------------|
| `visibility` | `string` | `VISIBLE`, `INVISIBLE`, `GONE` | Field visibility in fill mode |
| `viewModeVisibility` | `string` | `VISIBLE`, `INVISIBLE`, `GONE` | Visibility in view mode |
| `mandatory` | `boolean` | `true/false` | Field is required |
| `readOnly` | `boolean` | `true/false` | Field cannot be edited |
| `roles` | `string[]` | Role names | Which roles can see this field |
| `cdr` | `string[]` | CDR names | Company Defined Roles access |
| `userIds` | `UUID[]` | User IDs | Specific users access |
| `answer` | `any` | Default value | Pre-filled default value |
| `userProfileDriven` | `boolean` | `true/false` | Auto-fill from user profile |

#### Visibility Values

| Value | Description |
|-------|-------------|
| `VISIBLE` | Field is shown normally |
| `INVISIBLE` | Field is hidden but takes space (for layout) |
| `GONE` | Field is completely removed from layout |

**Example: Role-Based Visibility**
```json
{
  "adminNotes": {
    "title": "Admin Notes",
    "accessMatrix": {
      "visibility": "VISIBLE",
      "roles": ["ADMIN", "SUPERADMIN"],
      "readOnly": false
    }
  }
}
```

## Field-Specific Appearance

### Section Arrays

| Property | Type | Description |
|----------|------|-------------|
| `newRowLabel` | `string` | Label for "Add Row" button |
| `hideAddNewRowButton` | `boolean` | Hide the add button |
| `keysForRowHeader` | `string[]` | Fields to show in row header |
| `minRows` | `number` | Minimum required rows |
| `maxRows` | `number` | Maximum allowed rows |
| `immutableRowCount` | `boolean` | Prevent adding/removing rows |

```json
{
  "familyMembers": {
    "type": "array",
    "description": "section",
    "newRowLabel": "+ Add Family Member",
    "keysForRowHeader": ["name", "relation"],
    "minRows": 1,
    "maxRows": 10
  }
}
```

### Selection Fields

| Property | Type | Description |
|----------|------|-------------|
| `sortOptions` | `boolean` | Sort enum options alphabetically |
| `hideSearchIcon` | `boolean` | Hide search in dropdown |
| `selectAllButtonText` | `string` | Custom "Select All" text |
| `allowWildcardSelect` | `boolean` | Allow selecting all options |

```json
{
  "categories": {
    "description": "string_list",
    "type": "array",
    "sortOptions": true,
    "selectAllButtonText": "Choose All Categories",
    "enum": ["A", "B", "C", "D"]
  }
}
```

### Multimedia Fields

| Property | Type | Description |
|----------|------|-------------|
| `multimediaType` | `string` | `audio`, `video`, `image`, `document`, `canvas` |
| `disableFilepicker` | `boolean` | Force camera only |
| `enableGeoTagging` | `boolean` | Add location to media |
| `showGeoCoordinates` | `boolean` | Display GPS coordinates |
| `enforceCameraFlash` | `boolean` | Force flash on |
| `shouldNotAutoDismissCameraView` | `boolean` | Keep camera open after capture |
| `embedImageMetadata` | `boolean` | Embed EXIF data |

```json
{
  "sitePhoto": {
    "title": "Site Photo",
    "description": "multimedia",
    "multimediaType": "image",
    "enableGeoTagging": true,
    "showGeoCoordinates": true,
    "disableFilepicker": true,
    "cameraHint": "Capture the entire site"
  }
}
```

### Date/Time Fields

| Property | Type | Description |
|----------|------|-------------|
| `dateOnly` | `boolean` | Date picker only (no time) |
| `timeOnly` | `boolean` | Time picker only (no date) |
| `systemTime` | `boolean` | Auto-fill with system time |
| `format` | `string` | Display format (`dd-MM-yyyy`, `HH:mm`) |
| `enableEventScheduling` | `boolean` | Allow calendar scheduling |

```json
{
  "auditDate": {
    "title": "Audit Date",
    "description": "timestamp",
    "dateOnly": true,
    "format": "dd-MM-yyyy"
  }
}
```

### Number Fields

| Property | Type | Description |
|----------|------|-------------|
| `format` | `string` | Number format (`#.##`, `#,###`) |
| `minimum` | `number` | Minimum value |
| `maximum` | `number` | Maximum value |
| `layout` | `string` | Use `counter` for stepper |

```json
{
  "quantity": {
    "title": "Quantity",
    "description": "number",
    "layout": "counter",
    "minimum": 0,
    "maximum": 100
  }
}
```

### Location Fields

| Property | Type | Description |
|----------|------|-------------|
| `pickManually` | `boolean` | Allow manual location selection |
| `editable` | `boolean` | Allow editing after capture |
| `locationMandatory` | `boolean` | Location must be captured |

## Conditional Formatting

Apply dynamic styles based on field values using predicates.

### Using CONDITIONAL_FORMAT Predicate

```json
{
  "status": {
    "title": "Status",
    "type": "string",
    "predicates": [
      {
        "action": "CONDITIONAL_FORMAT",
        "actionConfig": {
          "conditionalFormatPredicateConfig": {
            "conditionalFormats": [
              {
                "condition": "this.status == 'Approved'",
                "fieldDisplayConfiguration": {
                  "backgroundColor": "#e8f5e9",
                  "textColor": "#2e7d32"
                }
              },
              {
                "condition": "this.status == 'Rejected'",
                "fieldDisplayConfiguration": {
                  "backgroundColor": "#ffebee",
                  "textColor": "#c62828"
                }
              }
            ]
          }
        }
      }
    ]
  }
}
```

### Status-Based Coloring (Form Level)

In `formSetting.formBubbleSetting.statusPreviewConfiguration`:

```json
{
  "formSetting": {
    "formBubbleSetting": {
      "statusPreviewConfiguration": {
        "Pending": {
          "fontColor": "#ff9800",
          "backgroundColor": "#fff3e0"
        },
        "Approved": {
          "fontColor": "#4caf50",
          "backgroundColor": "#e8f5e9"
        },
        "Rejected": {
          "fontColor": "#f44336",
          "backgroundColor": "#ffebee"
        }
      }
    }
  }
}
```

## Validation Display

### `validationMessage` üñ•Ô∏è

Custom error message when validation fails.

```json
{
  "email": {
    "title": "Email",
    "pattern": "^[a-zA-Z0-9+_.-]+@[a-zA-Z0-9.-]+$",
    "validationMessage": "Please enter a valid email address"
  }
}
```

### Validation with Custom Styling

Using VALIDATE predicate with custom error display:

```json
{
  "age": {
    "predicates": [
      {
        "condition": "this.age < 18",
        "action": "VALIDATE",
        "actionConfig": {
          "validatePredicateConfig": {
            "errorMessage": "Must be 18 or older",
            "resultLayout": "INLINE"
          }
        }
      }
    ]
  }
}
```
