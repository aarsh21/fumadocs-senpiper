## Layout Properties

### `layout` üñ•Ô∏è

Changes the visual representation of a field without changing its data type. Think of it as "skins" for fields.

```json
{
  "fieldKey": {
    "description": "string_list",
    "layout": "radio"
  }
}
```

#### Available Values by Field Type

| Field Type | Layout Options | Description |
|------------|---------------|-------------|
| **string_list** | `radio` | Radio buttons (single select) |
| | `checkbox` | Checkboxes (multi-select) |
| | `small_pill` / `smallPill` | Small pill/chip buttons ‚ö†Ô∏è |
| | `large_pill` / `largePill` | Large pill/chip buttons ‚ö†Ô∏è |
| | *(default)* | Dropdown picker |
| **section** | `newPage` | Renders section on a new screen |
| | `accordian` | Collapsible accordion |
| | `expandable` / `expanded` | Initially expanded |
| | `collapsible` / `collapsed` | Initially collapsed |
| | `table` | Table layout - used with section arrays; renders each section as a table row with titles as columns |
| **number** | `counter` | Stepper/counter UI with +/- buttons |

<Callout type="info">
**Section Layout Notes:**
- `inline` is **not** used with section type
- `book` and `tab` are not currently implemented but reserved for future use
- `card` layout has been removed (not supported)

**Label Fields:**
- Label fields do **not** support the `layout` property

**Pill Layouts (‚ö†Ô∏è):**
- `small_pill`/`smallPill` and `large_pill`/`largePill` - verify deprecation status before use
</Callout>

**Example: Radio Layout**
```json
{
  "gender": {
    "title": "Gender",
    "type": "string",
    "description": "string_list",
    "layout": "radio",
    "enum": ["Male", "Female", "Other"]
  }
}
```

**Example: Accordion Section**
```json
{
  "addressSection": {
    "title": "Address Details",
    "type": "object",
    "description": "section",
    "layout": "accordian",
    "properties": { ... }
  }
}
```

### `webLayout` üñ•Ô∏è

Web-specific layout override. **On web, when `webLayout` is present, the web client reads only the `webLayout` property and completely ignores the `layout` property.**

```json
{
  "items": {
    "description": "section",
    "type": "array",
    "layout": "newPage",
    "webLayout": "table"
  }
}
```

| Value | Description |
|-------|-------------|
| `table` | Render array section as table |
| `small_pill` | Pill selection on web |
| `large_pill` | Large pill selection |
| `accordian` | Accordion on web |
| `radio` | Radio buttons |
| `checkbox` | Checkboxes |

## Text & Label Properties

### `title` üñ•Ô∏è

The field's display label shown to users.

```json
{
  "firstName": {
    "title": "First Name",
    "type": "string"
  }
}
```

### `hint` üñ•Ô∏è

Tooltip or helper text shown near the field. Provides additional context.

```json
{
  "panNumber": {
    "title": "PAN Number",
    "hint": "Enter your 10-character PAN (e.g., ABCDE1234F)",
    "type": "string"
  }
}
```

### `inlineHint` üñ•Ô∏è

When `true`, displays the hint inline with the field instead of as a tooltip.

```json
{
  "email": {
    "title": "Email",
    "hint": "We'll send verification to this email",
    "inlineHint": true
  }
}
```

### `placeholder` üñ•Ô∏è

Placeholder text shown inside empty input fields.

```json
{
  "searchQuery": {
    "title": "Search",
    "placeholder": "Type to search...",
    "type": "string"
  }
}
```

### `prefixText` üñ•Ô∏è

Text displayed before the input (inside the field container).

<Callout type="info">
App and web teams should refer to visual guides for usage examples.
</Callout>

```json
{
  "price": {
    "title": "Price",
    "prefixText": "‚Çπ",
    "type": "number"
  }
}
```

### `suffixText` üñ•Ô∏è

Text displayed after the input (inside the field container).

<Callout type="info">
App and web teams should refer to visual guides for usage examples.
</Callout>

```json
{
  "weight": {
    "title": "Weight",
    "suffixText": "kg",
    "type": "number"
  }
}
```

### `wrapStyle` üñ•Ô∏è

Controls how text wraps within the field. **Web only: primarily used for viewing titles in tabular section layouts.**

| Value | Description |
|-------|-------------|
| `trim` | Truncate long text with ellipsis |
| `wrap` | Wrap text to multiple lines |

```json
{
  "description": {
    "title": "Description",
    "wrapStyle": "wrap",
    "type": "string"
  }
}
```

### `renderAsMd` üñ•Ô∏è

When `true`, renders text as Markdown. **Works on any field titles and label fields.**

```json
{
  "instructions": {
    "title": "**Important:** Please read carefully",
    "description": "label",
    "renderAsMd": true
  }
}
```

### `cameraHint` üñ•Ô∏è

Text displayed when camera is opened for multimedia fields.

```json
{
  "idCard": {
    "title": "ID Card Photo",
    "description": "multimedia",
    "cameraHint": "Position ID card within the frame"
  }
}
```

## Display Configuration Objects

These objects provide fine-grained control over field appearance in different modes.

### `titleDisplayConfiguration` üñ•Ô∏è

Controls how the field title/label is displayed.

```json
{
  "fieldKey": {
    "titleDisplayConfiguration": {
      "buttonConfiguration": {
        "localisationMap": {}
      },
      "backgroundColor": "#f5f5f5",
      "fontColor": "#333333",
      "fontSize": 14,
      "fontWeight": "bold"
    }
  }
}
```

### `viewModeConfiguration` üñ•Ô∏è

Controls appearance when form is in **view/read-only mode**.

```json
{
  "amount": {
    "viewModeConfiguration": {
      "backgroundColor": "#e8f5e9",
      "fontColor": "#2e7d32",
      "fontSize": 16,
      "fontWeight": "bold"
    }
  }
}
```

### `fillModeConfiguration` üñ•Ô∏è

Controls appearance when form is in **edit/fill mode**.

```json
{
  "criticalField": {
    "fillModeConfiguration": {
      "backgroundColor": "#fff3e0",
      "borderColor": "#ff9800",
      "borderWidth": 2
    }
  }
}
```

### `formDisplayConfiguration` üñ•Ô∏è

General display configuration for the field.

```json
{
  "highlighted": {
    "formDisplayConfiguration": {
      "backgroundColor": "#fffde7",
      "borderColor": "#fbc02d"
    }
  }
}
```

### `rowHeaderDisplayConfiguration` üñ•Ô∏è

For array sections, controls how row headers appear.

```json
{
  "items": {
    "type": "array",
    "rowHeaderDisplayConfiguration": {
      "backgroundColor": "#e3f2fd",
      "fontWeight": "bold"
    }
  }
}
```

### FieldDisplayConfiguration Properties

All display configuration objects support these properties:

| Property | Type | Description |
|----------|------|-------------|
| `backgroundColor` | `string` | Background color (hex: `#ffffff`) |
| `fontColor` / `textColor` | `string` | Text color (hex) |
| `borderColor` | `string` | Border color (hex) |
| `borderWidth` | `number` | Border width in pixels |
| `borderRadius` | `number` | Corner radius in pixels |
| `fontSize` | `number` | Font size in pixels |
| `fontWeight` | `string` | `normal`, `bold`, `100`-`900` |
| `fontStyle` | `string` | `normal`, `italic` |
| `padding` | `number` | Internal padding |
| `margin` | `number` | External margin |

### Complete FieldDisplayConfiguration Structure

The `fillModeConfiguration` and `viewModeConfiguration` objects support a rich set of nested configuration objects for fine-grained control over field appearance:

```typescript
class FieldDisplayConfiguration {
    hideMenuIcon?: boolean;
    voiceInputEnabled?: boolean;
    renderAsMd?: boolean;
    displayOrientation?: DisplayOrientationConfiguration;
    fieldContainerConfiguration?: ContainerConfiguration;
    titleConfiguration?: ContainerConfiguration;
    descriptionConfiguration?: ContainerConfiguration;
    errorConfiguration?: ContainerConfiguration;
    valueConfiguration?: ContainerConfiguration;
    fieldSelectedConfiguration?: ContainerConfiguration;
    fieldUnSelectedConfiguration?: ContainerConfiguration;
    addAnotherButtonConfiguration?: ContainerConfiguration;
}
```

#### FieldDisplayConfiguration Properties

| Property | Type | Description |
|----------|------|-------------|
| `hideMenuIcon` | `boolean` | Show/hide the 3-dot menu icon in fields |
| `voiceInputEnabled` | `boolean` | Show/hide the mic icon for voice input (text fields) |
| `renderAsMd` | `boolean` | Render field content as Markdown |
| `displayOrientation` | `DisplayOrientationConfiguration` | Controls title and value arrangement (side-by-side layouts) |
| `fieldContainerConfiguration` | `ContainerConfiguration` | Layout UI for the entire field container |
| `titleConfiguration` | `ContainerConfiguration` | Layout UI for the title/label container |
| `descriptionConfiguration` | `ContainerConfiguration` | Layout UI for the description container |
| `errorConfiguration` | `ContainerConfiguration` | Layout UI for the error message container |
| `valueConfiguration` | `ContainerConfiguration` | Layout UI for the value/input container |
| `fieldSelectedConfiguration` | `ContainerConfiguration` | Design for selected options (select/multiselect pills/checkboxes) |
| `fieldUnSelectedConfiguration` | `ContainerConfiguration` | Design for unselected options (select/multiselect pills/checkboxes) |
| `addAnotherButtonConfiguration` | `ContainerConfiguration` | Style for "Add Another" button in section arrays |

#### DisplayOrientationConfiguration

Controls how field titles and values are arranged, particularly for side-by-side layouts.

```typescript
class DisplayOrientationConfiguration {
    layout?: string;
    titleAndValueArrangement?: string;  // "default" or "sideBySide"
    titleHorizontalPercentage?: number; // 0-100
    spaceBetweenTitleAndValuePercentage?: number; // 0-100
}
```

**Example: Side-by-Side Layout**
```json
{
  "pincode": {
    "title": "Pincode",
    "type": "number",
    "fillModeConfiguration": {
      "displayOrientation": {
        "titleAndValueArrangement": "sideBySide",
        "titleHorizontalPercentage": 70,
        "spaceBetweenTitleAndValuePercentage": 10
      }
    }
  }
}
```

#### ContainerConfiguration

Defines the visual appearance of any container (field, title, value, etc.).

```typescript
class ContainerConfiguration {
    backgroundConfiguration?: BackgroundConfiguration;
    textConfiguration?: TextConfiguration;
    spacing?: MarginConfiguration;
}
```

#### BackgroundConfiguration

```typescript
class BackgroundConfiguration {
    borderThickness?: number;
    borderColor?: string;       // Hex code (e.g., "#EA4C89")
    borderRadius?: CornerRadius;
    color?: string;             // Background color hex code
    shadow?: string;
    imageName?: string;         // e.g., "field_background_9_patch"
    imageUrl?: string;
}
```

**Available Background Images:**
- `field_background_9_patch`
- `field_background_thicker_shadows_9_patch`

#### TextConfiguration

```typescript
class TextConfiguration {
    numberOfLines?: number;
    textAlignment?: string;     // "center", "left", "right", "top", "bottom"
    placeholderTextConfiguration?: TextConfiguration;
    selectAllTextConfiguration?: TextConfiguration;
    fontConfiguration?: FontConfiguration;
    prefixConfiguration?: ContainerConfiguration;
    suffixConfiguration?: ContainerConfiguration;
}
```

#### FontConfiguration

```typescript
class FontConfiguration {
    fontSize?: string;    // "extraSmall", "small", "medium", "large"
    fontWeight?: string;  // "thin", "light", "regular", "medium", "black"
    fontColor?: string;   // Hex code (e.g., "#EA4C89")
    fontStyle?: string;   // "italic", "regular", "strikethrough", "underline"
}
```

#### MarginConfiguration (Spacing)

```typescript
class MarginConfiguration {
    left?: number;
    top?: number;
    bottom?: number;
    right?: number;
}
```

**Example: Spacing Configuration**
```json
{
  "spacing": {
    "left": 10,
    "top": 10,
    "bottom": 10,
    "right": 10
  }
}
```

#### CornerRadius

```typescript
class CornerRadius {
    radius?: number;           // Uniform radius for all corners
    // OR specify individual corners:
    topLeftRadius?: number;
    topRightRadius?: number;
    bottomLeftRadius?: number;
    bottomRightRadius?: number;
}
```

<Callout type="info">
When using individual corner radii, you must specify **all four** values (topLeft, topRight, bottomLeft, bottomRight).
</Callout>

**Example: Uniform Radius**
```json
{
  "borderRadius": {
    "radius": 5
  }
}
```

**Example: Individual Corner Radii**
```json
{
  "borderRadius": {
    "topLeftRadius": 10,
    "topRightRadius": 10,
    "bottomLeftRadius": 0,
    "bottomRightRadius": 0
  }
}
```

## Visibility & Access

### `accessMatrix` ‚öôÔ∏è

Controls field visibility, editability, and requirements.

```json
{
  "sensitiveField": {
    "accessMatrix": {
      "visibility": "VISIBLE",
      "mandatory": true,
      "readOnly": false
    }
  }
}
```

#### AccessMatrix Properties

| Property | Type | Values | Description |
|----------|------|--------|-------------|
| `visibility` | `string` | `VISIBLE`, `INVISIBLE`, `GONE` | Field visibility in fill mode |
| `viewModeVisibility` | `string` | `VISIBLE`, `INVISIBLE`, `GONE` | Visibility in view mode |
| `mandatory` | `boolean` | `true/false` | Field is required |
| `readOnly` | `boolean` | `true/false` | Field cannot be edited |
| `roles` | `string[]` | Role names | Which roles can see this field ‚ö†Ô∏è |
| `cdr` | `string[]` | CDR names | Company Defined Roles access ‚ö†Ô∏è |
| `userIds` | `UUID[]` | User IDs | Specific users access ‚ö†Ô∏è |
| `answer` | `any` | Default value | Pre-filled default value |
| `userProfileDriven` | `boolean` | `true/false` | Auto-fill from user profile üìù |

<Callout type="info">
**Access Control Properties (‚ö†Ô∏è):**
The `roles`, `cdr`, and `userIds` properties are used **only** for workflow button accessMatrix objects, **not** for form field schema.

**userProfileDriven (üìù):**
This property is currently used directly in the form schema itself (not nested in accessMatrix). In future versions, this should be part of the accessMatrix object.
</Callout>

#### Visibility Values

| Value | Description |
|-------|-------------|
| `VISIBLE` | Field is shown normally |
| `INVISIBLE` | Field is hidden but takes space (for layout) |
| `GONE` | Field is completely removed from layout. **Note:** Fields with GONE visibility do not contribute to answers submitted or drafted on server and client. |

**Example: Role-Based Visibility**
```json
{
  "adminNotes": {
    "title": "Admin Notes",
    "accessMatrix": {
      "visibility": "VISIBLE",
      "roles": ["ADMIN", "SUPERADMIN"],
      "readOnly": false
    }
  }
}
```

## Conditional Formatting

Apply dynamic styles based on field values using predicates.

### Using CONDITIONAL_FORMAT Predicate

```json
{
  "status": {
    "title": "Status",
    "type": "string",
    "predicates": [
      {
        "action": "CONDITIONAL_FORMAT",
        "actionConfig": {
          "conditionalFormatPredicateConfig": {
            "conditionalFormats": [
              {
                "condition": "this.status == 'Approved'",
                "fieldDisplayConfiguration": {
                  "backgroundColor": "#e8f5e9",
                  "textColor": "#2e7d32"
                }
              },
              {
                "condition": "this.status == 'Rejected'",
                "fieldDisplayConfiguration": {
                  "backgroundColor": "#ffebee",
                  "textColor": "#c62828"
                }
              }
            ]
          }
        }
      }
    ]
  }
}
```

### Status-Based Coloring (Form Level)

In `formSetting.formBubbleSetting.statusPreviewConfiguration`:

```json
{
  "formSetting": {
    "formBubbleSetting": {
      "statusPreviewConfiguration": {
        "Pending": {
          "fontColor": "#ff9800",
          "backgroundColor": "#fff3e0"
        },
        "Approved": {
          "fontColor": "#4caf50",
          "backgroundColor": "#e8f5e9"
        },
        "Rejected": {
          "fontColor": "#f44336",
          "backgroundColor": "#ffebee"
        }
      }
    }
  }
}
```

## Validation Display

### `validationMessage` üñ•Ô∏è

Custom error message when validation fails.

```json
{
  "email": {
    "title": "Email",
    "pattern": "^[a-zA-Z0-9+_.-]+@[a-zA-Z0-9.-]+$",
    "validationMessage": "Please enter a valid email address"
  }
}
```

### Validation with Custom Styling

Using VALIDATE predicate with custom error display:

```json
{
  "age": {
    "predicates": [
      {
        "condition": "this.age < 18",
        "action": "VALIDATE",
        "actionConfig": {
          "validatePredicateConfig": {
            "errorMessage": "Must be 18 or older",
            "resultLayout": "INLINE"
          }
        }
      }
    ]
  }
}
```
